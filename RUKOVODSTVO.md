## 1. Что делает проект

Проект **news** — набор Python-скриптов, которые:

| Задача | Скрипт | Куда приходит результат |
|--------|--------|-------------------------|
| Собрать новости из VK, Telegram и RSS, убрать дубли, отранжировать через ИИ | `run_digest.py` | Два разных Telegram-бота (общий дайджест и ЧП) |
| Следить за новыми постами в VK-пабликах и готовить push-рассылки подписчикам | `rass.py` | Telegram (превью) → по кнопке рассылка во ВКонтакте |
| Переписать новость для паблика: заголовки, текст, вопросы, блок подписки VK | `rewrite.py` | Telegram-бот по запросу |

Все тексты для Gemini генерируются моделью **`gemini-3.1-flash-lite-preview`** (задана в `config.py`, менять только там).

## 2. Три сервиса — кратко

```
┌─────────────────────────────────────────────────────────────────┐
│  run_digest.py  (один процесс на компьютере)                    │
│  ├── профиль NCH → бот TG_BOT_TOKEN_NCH  (каждые 48 мин)        │
│  └── профиль CHP → бот TG_BOT_TOKEN_CHP  (каждые 55 мин)        │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  rass.py  — мониторинг стен VK → превью в TG → рассылка VK      │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  rewrite.py  — вы присылаете текст → бот отдаёт готовый пост    │
└─────────────────────────────────────────────────────────────────┘
```

**Важно:** дайджесты — **один процесс**, два бота. `rass.py` и `rewrite.py` — **отдельные процессы** (три окна терминала или три службы).

## 3. Требования и установка

### Что нужно на компьютере

- **Windows 10/11** (или Linux — команды аналогичны)
- **Python 3.10+** ([python.org](https://www.python.org/downloads/))
- Стабильный **интернет**
- Доступ к API: Google Gemini, Telegram, ВКонтакте, сервис рассылок VK

### Первичная установка

1. Скопируйте папку проекта, например `C:\Users\...\Desktop\news`.
2. Откройте терминал в этой папке.
3. (Рекомендуется) создайте виртуальное окружение:

```bash
python -m venv .venv
.venv\Scripts\activate
```

4. Установите зависимости:

```bash
pip install -r requirements.txt
```

Первая установка может занять **10–20 минут** из‑за библиотеки `sentence-transformers` (модель эмбеддингов для дайджестов).

5. Скопируйте шаблон секретов:

```bash
copy .env.example .env
```

6. Заполните `.env` (см. раздел 4). Без него скрипты не запустятся.

## 4. Настройка `.env`

Файл **`.env`** лежит в корне проекта. В git не попадает (см. `.gitignore`). Все ключи храните только там.

| Переменная | Для чего |
|------------|----------|
| `ADMIN_CHAT_ID` | Ваш числовой ID в Telegram — куда дайджесты и превью рассылок приходят личным сообщением |
| `VK_TOKEN` | Токен VK API — **чтение** постов со стен (`wall.get`) |
| `VK_COMMUNITY_TOKEN` | Токен **сообщества** с правами `photos` и `messages` — загрузка фото и рассылки в `rass.py` |
| `VK_BROADCAST_API_TOKEN` | Ключ API сервиса «Рассылки» ВК ([документация](https://vk.com/@hs-api-servisa-rassylki-soobschenii)) |
| `VK_BROADCAST_LIST_ID` | ID списка рассылки в приложении (например `1424009`) |
| `VK_BROADCAST_GROUP_ID` | ID сообщества **без минуса** (для `app5748831_-211836744` это `211836744`) |
| `GEMINI_API_KEY_NCH` | Ключ Google AI для общего дайджеста |
| `GEMINI_API_KEY_CHP` | Ключ Google AI для дайджеста ЧП |
| `GEMINI_API_KEY_RASS` | Ключ Google AI для push-текстов |
| `GEMINI_API_KEY_REWRITE` | Ключ Google AI для рерайта |
| `TG_BOT_TOKEN_NCH` | Токен @BotFather для бота общего дайджеста |
| `TG_BOT_TOKEN_CHP` | Токен бота дайджеста ЧП/происшествий |
| `TG_BOT_TOKEN_RASS` | Токен бота превью push-рассылок |
| `TG_BOT_TOKEN_REWRITE` | Токен бота рерайта |

### Как узнать `ADMIN_CHAT_ID`

1. Напишите боту [@userinfobot](https://t.me/userinfobot) в Telegram.
2. Скопируйте поле **Id** в `.env`.

### Где взять ключ API рассылок VK

1. Откройте приложение рассылок сообщества (например `https://vk.com/app5748831_-211836744`).
2. Управление сервисом → **Настройки** → **Создать ключ** (до 5 ключей).
3. ID списка рассылки: вкладка **Рассылки** → карандаш у нужной рассылки → поле **Id рассылки**.

- Не запускайте два экземпляра одного и того же скрипта — возможны дубли в Telegram.

## 6. Дайджесты (`run_digest.py`)

### Назначение

Автоматически собирает свежие материалы из настроенных источников, отфильтровывает рекламу и дубли, отправляет в Gemini список «нового», получает **рейтинг новостей** и присылает вам в Telegram.

### Два профиля в одном процессе

| Профиль | Файл | Интервал | Заголовок сообщения | Бот |
|---------|------|----------|---------------------|-----|
| **nch** | `profiles/nch.yaml` | 48 минут | 🏙 ЧЕЛЯБИНСКИЙ ДАЙДЖЕСТ | `TG_BOT_TOKEN_NCH` |
| **chp** | `profiles/chp.yaml` | 55 минут | 🚨 ДАЙДЖЕСТ ЧП И ПРОИСШЕСТВИЙ | `TG_BOT_TOKEN_CHP` |

Профили **разные**: у ЧП другие VK/TG-источники и другой промпт (фокус на происшествия, ДТП, криминал).

### Как проходит один цикл (пошагово)

1. **Парсинг** (параллельно): для каждого источника берётся до 7 последних записей:
   - Telegram — веб-страница `t.me/s/канал`;
   - VK — `wall.get` по короткому имени группы;
   - RSS — лента по URL.
2. **Фильтр рекламы** — отбрасываются тексты с `erid`, «реклама», «партнёр» и т.п.
3. **Дедупликация**:
   - по **ID поста** (таблица `seen_posts` в SQLite);
   - по **смыслу** — эмбеддинги `rubert-tiny2`, порог схожести 0.83 (настраивается в YAML).
4. **Кластеризация** — похожие новости схлопываются; в Gemini уходит не больше **30** представителей за цикл.
5. **Gemini** — полные промпты из `profiles/prompts/`; ответ — готовый текст рейтинга (место, заголовок, суть, резонанс X/10, ссылка).
6. **Отправка** в `ADMIN_CHAT_ID` через соответствующего Telegram-бота.

Если новых новостей нет — сообщение не отправляется (в логе: «Новых новостей нет»).

### Что вы видите в Telegram

Пример структуры:

```
🏙 ЧЕЛЯБИНСКИЙ ДАЙДЖЕСТ (14:30)
--------------------------------

1 место — ...

...суть...

Резонанс: 9/10

Ссылка: https://...

2 место — ...
```

Обоснований «почему вошло в рейтинг» в промпте **нет** — только факты и резонанс.

### Логи дайджеста

Файл: **`data/digest.log`** — то же, что в консоли.

### Базы данных

| Файл | Профиль |
|------|---------|
| `news_db.sqlite` | nch |
| `news_db_chp.sqlite` | chp |

Хранят векторы и ID уже виденных постов (~15 часов для сравнения по смыслу).

## 7. Push-рассылки VK (`rass.py`)

### Назначение

Следит за **новыми постами** на стенах указанных VK-пабликов. Для каждого нового поста **с фото** (не реклама) генерирует короткий push-текст, присылает **превью в Telegram**. Вы решаете: отправить рассылку подписчикам, изменить текст или отменить.

### Какие паблики мониторятся

Список в начале файла `rass.py`, переменная **`VK_DOMAINS`**, например:

```python
VK_DOMAINS = ["nashchelyabinsk", "chp174", "proverkadimas"]
```

Добавление: вписать короткое имя паблика (как в URL `vk.com/имя`) и перезапустить скрипт.

### Поведение при старте

При каждом запуске бот запоминает **последний post_id** на стене каждого паблика и **не шлёт** старые посты. Уведомления только о постах, опубликованных **после** запуска (или после предыдущего цикла).

Проверка стены — каждые **60 секунд** (`CHECK_INTERVAL`).

### Что пропускается автоматически

- Посты с маркером **erid** (реклама).
- Посты **без фото** (для видео берётся превью кадра, если есть).

### Текст рассылки (формат)

```
{push-текст от Gemini}. Подробнее: https://vk.com/wall{owner_id}_{post_id}

📝 Управление рассылками: https://vk.com/app5748831_{-ID_сообщества}
```

Ссылка на управление строится из **`VK_BROADCAST_GROUP_ID`** в `.env`.

### Превью в Telegram

Приходит **фото поста** и подпись с полным текстом будущей рассылки и строкой «Выберите действие».

Три кнопки:

| Кнопка | Действие |
|--------|----------|
| **Отправить** | Скачать фото → загрузить на сервер VK → запустить рассылку по списку `VK_BROADCAST_LIST_ID` |
| **Изменить текст рассылки** | Бот просит прислать **одним сообщением** только новую фразу до «Подробнее» (без ссылки и без второго абзаца). Превью обновится. |
| **Отменить** | Черновик удаляется, кнопки убираются, в подписи появится «❌ Рассылка отменена» |

### Важно про рассылку

- Уходит **только с фото**. Без успешной загрузки на `pu.vk.com` рассылка **не создаётся**.
- Фото — **максимальный размер** с поста VK, без сжатия в боте.
- При ошибке загрузки (например **504 Gateway Time-out**) — до **4 попыток**. Если не вышло — сообщение об ошибке; можно нажать **Отправить** снова позже.
- После успешной отправки приходит: `✅ Рассылка с фото запущена, ID …`

### Техническая цепочка «Отправить»

1. `photos.getMessagesUploadServer` (токен `VK_COMMUNITY_TOKEN`)
2. POST файла на `upload_url` (сервер `pu.vk.com`)
3. `photos.saveMessagesPhoto` → attachment `photo-211836744_…`
4. POST в `https://broadcast.vkforms.ru/api/v2/broadcast` с `list_ids`, `run_now: 1`

### Состояние

Файл **`data/rass_state.json`** — последние обработанные `post_id` по каждому домену.

### Логи

Вывод в консоль (уровень INFO): скачивание фото (размер в KB), успех загрузки, ID рассылки, ошибки.

## 8. Рерайт постов (`rewrite.py`)

### Назначение

Интерактивный Telegram-бот: вы присылаете **исходный текст новости**, получаете варианты для публикации в VK-паблике.

### Как пользоваться

1. Запустите `python rewrite.py`.
2. Откройте бота с токеном `TG_BOT_TOKEN_REWRITE` в Telegram.
3. Команда `/start` — краткая подсказка.
4. **Отправьте текст новости** одним сообщением.
5. Дождитесь ответа «Генерирую варианты…» (обычно 10–30 секунд).

### Что генерирует бот

| Блок | Описание |
|------|----------|
| **5 заголовков** | До ~70 символов, с эмодзи в начале, без двоеточий |
| **Текст новости** | Два абзаца, живой язык |
| **5 вопросов** | Для вовлечения подписчиков, с ❓ |
| **Рассылка** | Строка подписки VK из фиксированного списка (модель выбирает ID) |

### Кнопки после генерации

Первая строка: **1–5** — выбор заголовка.  
Вторая строка: **1–5** — выбор вопроса, **0** — без вопроса.  
**✅ Готово** — собрать финальный пост.

Порядок: сначала выберите заголовок (обязательно), при желании вопрос, затем **Готово**.

### Финальный пост

Бот пришлёт готовый текст и сохранит копию в:

**`data/rewrites/ГГГГ-ММ-ДД_ЧЧММСС.txt`**

### Список рассылок VK (whitelist)

Зашит в `rewrite.py` (`NEWSLETTERS`). Модель возвращает только **ID**; бот подставляет готовую строку с ссылкой — ссылки не «придумываются» заново.

Темы: хорошие новости, ЖКХ, происшествия, экология, важные, спорт, катаклизмы, афиша, политика/экономика, ХК «Трактор».

## 9. Настройка источников и промптов

### Профили дайджеста (`profiles/*.yaml`)

| Параметр | Смысл |
|----------|--------|
| `schedule_minutes` | Как часто запускать цикл |
| `digest_header` | Первая строка сообщения в Telegram |
| `similarity_threshold` | Порог дубля по смыслу (0.83) |
| `cluster_threshold` | Порог объединения похожих перед Gemini (0.75) |
| `min_text_length` | Минимум символов в новости |
| `posts_per_source` | Сколько записей брать с каждого источника |
| `max_gemini_items` | Максимум новостей в одном запросе к Gemini |
| `sources.tg_channels` | Имена TG-каналов без `@` |
| `sources.vk_groups` | Короткие имена групп VK |
| `sources.rss_feeds` | URL RSS-лент |

После правки YAML перезапустите `run_digest.py`.

### Промпты Gemini

| Файл | Профиль |
|------|---------|
| `profiles/prompts/nch_system.txt` | Системная роль общего дайджеста |
| `profiles/prompts/nch_user.txt` | Задание + `{news_content}` |
| `profiles/prompts/chp_system.txt` | Системная роль ЧП |
| `profiles/prompts/chp_user.txt` | Задание ЧП |

Плейсхолдер **`{news_content}`** подставляется автоматически — не удаляйте его из `*_user.txt`.

### Push-текст (`rass.py`)

Шаблон промпта — константа **`PROMPT_TEMPLATE`** в начале `rass.py`. Редактируется вручную; перезапуск `rass.py` обязателен.

## 11. Устранение неполадок

### Дайджесты не приходят

- Проверьте, запущен ли `run_digest.py` и нет ли ошибок в `data/digest.log`.
- Убедитесь, что за цикл появились **новые** новости (не дубли).
- Проверьте `ADMIN_CHAT_ID` и токены ботов NCH/CHP.
- Ошибки Gemini 503 — бот ждёт и повторяет; при долгой недоступности придёт `⚠️ Ошибка бота …`.

### Rass: старые посты при старте

Не должны приходить — при старте фиксируется baseline. Если приходят — проверьте, один ли экземпляр `rass.py` запущен.

### Rass: 504 при загрузке фото

Временная перегрузка `pu.vk.com` или тяжёлый файл (в логе «Скачано фото: N KB»). Подождите 1–2 минуты и нажмите **Отправить** снова. Рассылка **без фото** не отправляется намеренно.

### Rass: рассылка ушла, но ошибка в Telegram

Бывает, если не удалось убрать кнопки с превью (баг API Telegram). На рассылку это не влияет — смотрите ID в логе «Рассылка создана».

### Rewrite: «Ошибка формата»

Gemini вернул не JSON — отправьте текст ещё раз или упростите исходник.

### «Переменная окружения X не задана»

Заполните поле в `.env`, сохраните файл, перезапустите скрипт из корня проекта.

### Нет модуля / ImportError

```bash
pip install -r requirements.txt
```

## 12. Безопасность

- **Не выкладывайте** `.env` в чат, почту и git.
- Токены VK и Telegram при утечке — **отозвать и выпустить новые**.
- Ключ API рассылок — в настройках приложения VK (до 5 ключей, можно удалить скомпрометированный).
- Доступ к ботам: только сотрудники с `ADMIN_CHAT_ID` для служебных ботов; rewrite-бот — кто знает ссылку на бота.

## Быстрая шпаргалка для нового сотрудника

1. Установить Python, `pip install -r requirements.txt`, заполнить `.env`.
2. Запустить три скрипта в трёх терминалах (или настроить автозапуск).
3. **Дайджесты** — ждать сообщения в двух ботах; источники и промпты — в `profiles/`.
4. **Push** — смотреть бота Rass; на каждый пост: Отправить / Изменить текст / Отменить.
5. **Рерайт** — прислать новость боту Rewrite → кнопки 1–5, 0, Готово.
6. Проблемы — `data/digest.log`, консоль `rass.py`, раздел 11 этого руководства.
