# Новостные боты

**Полное руководство для коллег:** [RUKOVODSTVO.md](RUKOVODSTVO.md) — установка, все три сервиса, кнопки, настройки, ошибки.

## Запуск

```bash
pip install -r requirements.txt
```

Скопируйте `.env.example` в `.env` и заполните ключи (в `.env` уже перенесены значения из старых скриптов).

| Команда | Назначение |
|---------|------------|
| `python run_digest.py` | Оба дайджеста в одном процессе (разные Telegram-боты) |
| `python rass.py` | Push: превью в Telegram, кнопка «Отправить» → рассылка VK |
| `python rewrite.py` | Рерайт новостей по запросу в Telegram |

## Модель Gemini

Во всех скриптах используется одна модель: **`gemini-3.1-flash-lite-preview`** (константа `GEMINI_MODEL` в `config.py`).

## Структура

- `core/` — парсинг, дедуп, Gemini, отправка
- `profiles/nch.yaml`, `profiles/chp.yaml` — источники и интервалы
- `profiles/prompts/` — промпты для редакторской логики
- `data/` — логи и состояние `rass`

## Настройка профилей

Источники и интервал — в `profiles/*.yaml`. Промпты — в `profiles/prompts/`.

Рекомендуется перевыпустить все токены, которые раньше лежали в коде, и обновить `.env`.

### Rass и рассылки VK

При нажатии «Отправить» фото **скачивается** с CDN ВК и **загружается** через `photos.getMessagesUploadServer` → `photos.saveMessagesPhoto`, затем в рассылку уходит `attachment` вида `photo-211836744_…`.

В `.env` нужны `VK_BROADCAST_GROUP_ID` (ID сообщества без минуса) и **`VK_COMMUNITY_TOKEN`** — токен сообщества с правами **photos** и **messages** (для загрузки фото и рассылки). `VK_TOKEN` по-прежнему используется только для чтения постов со стены.

Фото загружается в **исходном размере** (максимальное качество с поста). Рассылка уходит **только с фото**; в тексте второй абзац со ссылкой на управление рассылками (`app5748831_-{id сообщества}`). При ошибке загрузки после 4 попыток в Telegram приходит сообщение об ошибке — можно нажать «Отправить» снова.
